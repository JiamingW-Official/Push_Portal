# Push GTM & Growth — Overview

## Launch Philosophy
Concentrate on one small zone. One neighborhood, ~10 merchants, ~30 curated creators. Prove the loop before expanding. **Never launch "a city" — launch a ZIP code.**

## Beachhead Wedge: NYC Café + Dessert + Beauty
**Why these categories:** high frequency, visual/photogenic, creator-dense, manageable complexity, merchant openness to trying new things

### Zone Selection Criteria
- 30+ independent F&B shops within 15-min walking radius
- Not chain-dominated
- Active local food/lifestyle creator scene on Instagram
- Walkable and photogenic

### Recommended Zones
- NYC: Williamsburg, LES, Astoria
- LA: Silver Lake, Echo Park, Sawtelle
- SF: Mission District, Hayes Valley

## Three Launch Loops

| Loop | Mechanism |
|------|-----------|
| **Merchant Outcome** | Better outcomes → trust → repeat → density → opportunity flow |
| **Creator Progression** | Better completion → score → better access → higher payout → retention |
| **Data Advantage** | More campaigns → better matching → higher repeat → stronger platform |

## Four Early Experiments
1. **Merchant Pain Discovery:** Which use case is strongest? (new visits vs off-peak vs launch vs event)
2. **Creator Supply Quality:** Which cohort is most reliable? (by follower count, category, platform)
3. **Standby Utility:** Does real-time refill change engagement levels?
4. **Merchant Repeat Trigger:** What drives the second campaign? (ROI dashboard? reminder? template?)

## Week 12 Go/No-Go Targets

| Metric | Target | Interpretation |
|--------|--------|-----------------|
| Merchant repeat rate | 40%+ run 2+ campaigns | ✅ GO |
| Creator completion rate | 85%+ | ✅ GO |
| Time-to-fill | < 48 hours | ✅ GO |
| Willingness to pay | 30%+ convert to paid | ✅ GO |
| NPS | > 7/10 both sides | ✅ GO |
| Total campaigns | 25+ | ✅ GO |
| Revenue | Any (proves WTP) | ✅ GO |

**Decision Logic:**
- ✅ GO: repeat > 30%, completion > 80%, some WTP
- ⚠️ ITERATE: one metric weak, others strong
- ❌ PIVOT: no repeat, unreliable creators, zero WTP

## Expansion Triggers
Only expand to new zone when current zone shows:
- Merchant repeat > 40%
- Creator completion > 85%
- Time-to-fill < 48h
- Paying merchants > 5
- Then replicate the exact same playbook in next ZIP code

## Retention Design

### Merchant Side
- Campaign templates (cognitive ease)
- One-click repeat (habit loop)
- Real-time dashboard (control illusion)
- Creator badges visible (social proof)
- ROI comparisons vs alternatives (anchoring)

### Creator Side
- Limited seats (FOMO)
- Standby queue (near-miss)
- Tier progression (growth narrative)
- Fast payouts (instant gratification)
- Non-portable score (loss aversion)
