## The Skill System Explained

Push uses a **tiered, specialized skill architecture** to help users navigate different domains of the platform.

### Why Skills Instead of One Monolithic Guide?

1. **Specialization:** Each skill has detailed operational guidelines for a specific domain (e.g., fraud ops, campaign design, GTM)
2. **Cross-referencing:** Skills point to the authoritative strategy documents (01-10) rather than duplicating content
3. **Scalability:** As Push evolves, skills can be updated independently without affecting the entire system
4. **Routing:** The `push-hub` skill acts as a master dispatcher—start here when you don't know which skill you need

### How Skills Map to Documents

Each skill is anchored in one or more authoritative documents. For example:

- **`push-campaign-designer`** → Primarily Doc 04 (Campaign Design Framework), with cross-refs to Doc 02 (Workflows) and Doc 04 (Merchant Goal Mapping)
- **`push-fraud-ops`** → Primarily Doc 06 (Fraud Detection), with cross-refs to Doc 02 (Disputes) and Doc 05 (Verification)
- **`push-gtm-experiments`** → Primarily Doc 07 (Go-to-Market), with cross-refs to Doc 09 (Metrics & Dashboards)

### The Three-Layer Workflow

When working on Push:

1. **Identify your task:** "I need to design a campaign template" or "I need to detect fraud"
2. **Start at the hub:** Use `push-hub` to find the right specialized skill
3. **Read the skill:** The skill's SKILL.md tells you exactly which documents to consult and how to approach the task
4. **Reference the source docs:** The authoritative documents (01-10) provide the detailed rules, frameworks, and logic

### Workflow: Example

**Scenario:** "We need to design a new creator tier to improve retention."

1. Start at `push-hub` → Find that `push-retention-architect` and `push-creator-scoring` are relevant
2. Read `push-retention-architect` SKILL.md → Points you to Doc 07 (GTM) and Doc 03 (Scoring)
3. Consult **Doc 03** (Creator Scoring & Progression System) → Learn the current tier mechanics
4. Reference **Doc 07** (Go-to-Market Strategy) → Understand how tiers drive retention
5. Use `push-product-spec` to write the feature specification grounded in both documents
